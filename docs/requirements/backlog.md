# Requirements Backlog

Use this file for candidate requirements discovered in legacy documents that are
not yet promoted into a canonical `specs/*` feature.

## Intake Rules

- Do not implement backlog items directly.
- Promote a backlog item by creating or updating a feature spec under `specs/`.
- Link the legacy source and explain why the item still matters.
- Remove or mark a backlog item `superseded` once it is covered by a canonical
  spec.

## Candidate Items

No candidate items were promoted during the documentation cleanup. Future audits
of `docs/modules/*`, `PERSISTENCE_REQUIREMENTS.md`, or `ANALYSIS_REPORT.md`
should add unresolved requirements here before implementation.
