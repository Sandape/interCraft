# Template Gallery Manifest Contract

**File**: `frontend/public/templates/manifest.json`
**Endpoint**: `GET /api/v1/v2/templates` (returns same payload)

Each template is described by a manifest consumed by:

1. The Template Gallery modal (US2): grid of thumbnails, name, description, tags.
2. The Template Settings panel (US3): current-template badge + "Swap" CTA.
3. The Template Marketplace filter (US13): industry + style filters.

---

## 1. Top-level shape

```json
{
  "$schema": "./template-manifest.schema.json",
  "version": 2,
  "updated_at": "2026-06-25T00:00:00Z",
  "templates": [ /* TemplateEntry, … exactly 10 entries */ ]
}
```

`version` increments on any breaking change (rename, removed template).
`updated_at` is ISO-8601 UTC.

---

## 2. TemplateEntry

```ts
interface TemplateEntry {
  id: TemplateId                            // one of 10 enum values
  name: string                              // English / Chinese display name
  description: string                       // 1-2 sentences; max 240 chars
  tags: string[]                            // 3..5 tags; max 8 chars each
  category: TemplateCategory                // enum, see §3
  thumbnail: string                         // path under /public/templates/jpg/<id>.jpg
  sidebar: SidebarPosition                  // see §4
  recommendedPage: "a4" | "letter"
  recommendedColors: { primary: string; text: string; background: string }
  defaults: ResumeDataV2                    // partial; only fields that differ from `defaultResumeDataV2()`
}
```

### 2.1 Example (Pikachu)

```json
{
  "id": "pikachu",
  "name": "Pikachu",
  "description": "彩色 header 卡片 + 左 sidebar，适合创意/设计/前端岗位",
  "tags": ["Two-column", "Creative", "Tech", "Visual flair"],
  "category": "creative",
  "thumbnail": "/templates/jpg/pikachu.jpg",
  "sidebar": "left",
  "recommendedPage": "a4",
  "recommendedColors": {
    "primary": "rgba(255, 200, 55, 1)",
    "text":    "rgba(31, 41, 55, 1)",
    "background": "rgba(255, 255, 255, 1)"
  },
  "defaults": {
    "metadata": {
      "design": { "colors": { "primary": "rgba(255,200,55,1)" } }
    }
  }
}
```

## 3. Categories

```ts
type TemplateCategory =
  | "minimal"        // onyx, kakuna
  | "business"       // azurill, bronzor, ditgar, rhyhorn
  | "creative"       // chikorita, pikachu, lapras
  | "editorial"      // scizor
```

Categories are non-overlapping. The Template Gallery groups by category with
tabs; the marketplace filters by category.

---

## 4. Sidebar positions

```ts
type SidebarPosition =
  | "left"           // onyx, azurill, ditgar, pikachu, lapras
  | "right"          // chikorita
  | "none"           // onyx (when fullWidth), kakuna, bronzor, scizor, rhyhorn
```

When `sidebar = "none"`, the template MUST NOT reference any sidebar layout —
the editor silently moves all sections into `main` and sets `sidebar = []`
across all pages.

---

## 5. Defaults override semantics

`defaults` is a **partial** `ResumeDataV2` overlaid on
`defaultResumeDataV2()` via `Object.assign`-style deep merge for objects.
Arrays are replaced, not merged (e.g., `metadata.styleRules: []` clears the
default).

Validation at load time:

- `id` MUST be unique within the manifest.
- `id` MUST be one of the 10 enum values.
- `tags.length` MUST be 3..5.
- `defaults` MUST round-trip through Zod schema (any failure = manifest load error).

Thumbnails are A4-rendered screenshots at 400×565 px. They are committed
to the repo and regenerated by `scripts/render-template-thumbnails.ts`
(Playwright + 027's unified renderer) when a template's CSS changes.

## 7. Marketplace compatibility

`Square.tsx` (existing v1 marketplace) loads `/data/template.json`. The v2
marketplace route loads `/api/v1/v2/templates` which returns the same shape
as `manifest.json`. v1 marketplace items can be imported into v2 with a
**fallback path**:

```json
{
  "v1_template_item": {
    "id": 17,
    "title": "Pikachu-like",
    "template": "..." /* v1 markdown */,
    "theme": "default",
    "v2_template": "pikachu",
    "v2_defaults": { /* partial ResumeDataV2 */ }
  }
}
```

The marketplace UI shows both v1 and v2 items with a "格式" toggle. Selecting
a v2 item POSTs `POST /api/v1/v2/resumes` with `from_sample=true` and the
included `v2_defaults` overlay.
