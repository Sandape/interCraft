# Avatar API Contract

**Date**: 2026-06-16 | **Feature**: 013-user-avatar | **Spec**: [spec.md](../spec.md)

## Auth

All endpoints require a valid bearer access token in the `Authorization` header, decoded by the existing `get_current_user` dependency. The authenticated user is the avatar owner; no cross-user reads are allowed.

## Common error envelope

All non-2xx responses use the project-standard envelope:

```json
{
  "error": "STABLE_ERROR_CODE",
  "message": "Human-readable, locale-friendly text",
  "request_id": "uuid"
}
```

`request_id` is taken from the `X-Request-ID` request header when present; otherwise it is generated by the server.

## POST `/api/v1/users/me/avatar`

Upload (or replace) the caller's avatar.

### Request

- Headers:
  - `Authorization: Bearer <access token>` (required)
  - `Content-Type: multipart/form-data` (required)
  - `X-Request-ID: <uuid>` (optional, surfaces in logs and error envelope)
- Body: a single `multipart/form-data` part named `file` with the image bytes.
  - Field: `file`
  - Type: `image/jpeg` or `image/png`
  - Max size: 2 MB (pre-checked by the router)
  - Max dimensions: 2048 px on any side (post-decode check)

### 200 OK — upload success

```json
{
  "avatar_id": "018f3b6a-2b4c-7c8a-9d1e-2f3a4b5c6d7e",
  "url": "/api/v1/users/me/avatar/018f3b6a-2b4c-7c8a-9d1e-2f3a4b5c6d7e",
  "content_type": "image/png",
  "byte_size": 17421,
  "width": 256,
  "height": 256,
  "created_at": "2026-06-16T08:00:00Z"
}
```

The `users.avatar_id` column is updated in the same transaction. The previous avatar file is left on disk for a future cleanup sweep; it is not returned.

### 400 Bad Request — empty body

```json
{ "error": "EMPTY_FILE", "message": "未选择文件", "request_id": "..." }
```

### 413 Payload Too Large — over size cap

```json
{ "error": "FILE_TOO_LARGE", "message": "图片不能超过 2MB", "request_id": "..." }
```

### 415 Unsupported Media Type — wrong MIME or content sniff mismatch

```json
{ "error": "UNSUPPORTED_FORMAT", "message": "仅支持 JPG / PNG 图片", "request_id": "..." }
```

### 422 Unprocessable Entity — over-dimension

```json
{ "error": "DIMENSION_TOO_LARGE", "message": "图片尺寸不能超过 2048×2048", "request_id": "..." }
```

### 500 Internal Server Error — storage write failed

```json
{ "error": "STORAGE_WRITE_FAILED", "message": "头像保存失败，请稍后重试", "request_id": "..." }
```

The previous avatar is preserved in this case.

## GET `/api/v1/users/me/avatar/{avatar_id}`

Fetch the bytes of an avatar owned by the caller. Streams from disk; the auth check enforces owner-only access (FR-006).

### Request

- Headers:
  - `Authorization: Bearer <access token>` (required)
- Path params:
  - `avatar_id` (UUID) — the avatar to fetch

### 200 OK — bytes

- Headers:
  - `Content-Type: image/jpeg` or `image/png` (matches `user_avatars.content_type`)
  - `Cache-Control: private, max-age=3600`
  - `Content-Length: <byte size>`
  - `X-Request-ID: <uuid>`
- Body: the avatar bytes (streamed, not buffered).

### 401 Unauthorized — no/invalid token

```json
{ "error": "UNAUTHORIZED", "message": "请先登录", "request_id": "..." }
```

### 404 Not Found — no such avatar, or not owned by the caller

```json
{ "error": "AVATAR_NOT_FOUND", "message": "头像不存在", "request_id": "..." }
```

The endpoint returns 404 (not 403) on cross-tenant access to avoid revealing whether the avatar exists for another user.

---

## DELETE `/api/v1/users/me/avatar`

Remove the caller's currently active avatar. Sets `users.avatar_id` to `NULL`. The underlying file is left on disk for the cleanup sweep.

### Request

- Headers:
  - `Authorization: Bearer <access token>` (required)

### 200 OK

```json
{
  "status": "removed",
  "message": "已移除头像"
}
```

### 404 Not Found — caller has no avatar

```json
{ "error": "AVATAR_NOT_FOUND", "message": "当前没有头像", "request_id": "..." }
```

## `users/me` payload (additive change)

The existing `GET /api/v1/users/me` response gains one optional field. No client is forced to consume it; clients that ignore unknown fields continue to work.

```diff
 {
   "id": "...",
   "email": "...",
   "display_name": "...",
   ...
+  "avatar_url": "/api/v1/users/me/avatar/018f3b6a-2b4c-7c8a-9d1e-2f3a4b5c6d7e" | null
 }
```

`avatar_url` is `null` when the user has no avatar set.

## Rate limit

All avatar endpoints share the `business` rate limit scope (enforced by `app.core.rate_limit.enforce_rate_limit`). No new rate limit bucket is added.
