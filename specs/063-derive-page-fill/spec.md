# Feature Specification: 派生简历满页校准与真实页数一致

**Feature Branch**: `[063-derive-page-fill]`

**Created**: 2026-07-12

**Status**: Ready for Tasks

**Input**: User description: "派生简历选择目标 N 页后，最终应得到满满当当的 N 页；纠正列表「目标/实际页数」与打开后真实分页不一致；用真实渲染测量与末页填充率决定先调行距还是 Agent 剪枝/扩写；保存时回写页数；导出仍以 PDF 页数为终裁。编号请求原为 REQ-062，因 062 已占用商业付费而落为 REQ-063。"

## Product Problem & Scope

REQ-055 已交付「一键派生 + 目标页数硬约束」产品承诺，但当前「实际页数」主要来自字符长度估算，与编辑器真实 A4 分页、导出 PDF 页数经常不一致。用户在简历中心看到「实际 1 页」，打开后却是两页；即便页数碰巧正确，末页也可能大片留白，达不到「选了几页就该是充实的几页」的投递观感。

本特性在不改变根简历/派生模型主叙事的前提下，把页数校准从「估算」升级为「真实测量 + 填充质量门禁」：目标 N 页时，成功派生稿必须页数严格等于 N，且末页内容充实；无法在不编造的前提下达标时进入引导态，而不是假成功。

### In Scope

- 派生过程中用与编辑器预览等价的真实分页测量页数与末页填充率。
- 按「溢出/不足量 + 末页填充率」决定：先调行距，再 Agent 剪枝或扩写，或直接引导/记 Bad Case。
- 派生成功落库时写入与测量一致的实际页数；失败不得伪报「实际=目标」。
- 编辑派生简历时，在分页变化并保存（或防抖持久化）后回写实际页数，修复存量不一致。
- 简历中心/岗位详情列表的「目标 x 页 · 实际 x 页」与打开后预览一致。
- 导出 PDF 页数硬门禁仍为最终权威；预览与 PDF 冲突时拦截并回写真实 PDF 页数。
- 校准决策与稀疏末页等 Bad Case 可被记录，供后续评测与预算调参。

### Non-Goals

- 不重新设计根简历素材库、一键派生选岗/主题向导、建议面板信息架构（沿用 REQ-055/059）。
- 不引入编造经历或无来源事实以「灌满」页面。
- 不要求每页物理填充率精确到 100%；「满」以可配置充实阈值判定（默认末页 ≥ 2/3）。
- 不在本特性内重做 Word 导出页数验收（仍以 PDF 为准）。
- 不替代 REQ-062（商业套餐与微信支付）。

### Governance Profile

**Highest Risk Class**: R2。本特性会改写用户派生简历正文、调用模型做剪枝/扩写，并将简历内容发往模型；不涉及跨租户权限变更或财务结算。

| Governed operation / effect | Risk class | Actor / target / trust boundary | Required authorization and evidence |
|---|---|---|---|
| 只读分页测量、填充率计算、列表展示页数 | R0 | 当前用户 / 本人简历 | 身份与资源所有权校验 |
| 行距在可读区间内自动调整、actual_page_count 回写 | R1 | 当前用户 / 本人派生简历 | 派生任务或编辑会话授权；可回滚到调整前行距 |
| Agent 剪枝/扩写正文、校准多轮自动执行、Bad Case 落库 | R2 | 当前用户 / 本人派生 + 模型处理边界 | 派生任务 standing authorization；禁止跨用户；事实须带来源；预算与轮次上限 |

| Dimension | Applicability, boundary, owner, or explicit N/A rationale |
|---|---|
| API / contract surface | 派生运行态、页数报告、导出门禁与列表字段语义在范围内；错误码需区分测量失败、不可行、页数不匹配 |
| Data and privacy | 派生正文与 page_report/Bad Case 属用户私有简历数据；租户隔离继承 resumes_v2 / resume_derive |
| Background / scheduled execution | 校准发生在既有派生后台任务内；需可取消、有轮次上限、可观测 |
| External side effects | 模型调用为可计费外部副作用；无渠道投递 |
| AI / Agent behavior | 剪枝/扩写必须遵守反编造与来源标注；畸形输出不得写入正文 |
| Schema / checkpoint migration | 可扩展 page_report / 填充率字段；不得破坏既有 derived 行可读性 |
| Operational release unit | 继承 resume_derive / resume 导出能力单元；本特性增加页数充实门禁 |

## User Scenarios & Testing *(mandatory)*

### User Story 1 — 目标 N 页得到充实的 N 页派生稿 (Priority: P1)

作为求职者，我选择目标页数 N 完成一键派生后，希望打开与导出的简历都是内容充实的 N 页，而不是空荡或页数对不上的稿件。

**Why this priority**: 这是本特性的核心产品承诺，直接修复当前「实际页数」失真与「半页空洞」问题。

**Independent Test**: 对同一根简历与岗位，分别选择目标 1/2/3 页完成派生；成功态下预览页数=N、末页充实、列表实际页数=N；导出 PDF 页数=N。

**Acceptance Scenarios**:

1. **Given** 用户选择目标页数为 N（N∈{1,2,3}）且根简历素材充足，**When** 派生自动校准完成，**Then** 预览实际页数等于 N，且末页填充率不低于充实阈值（默认 2/3），状态为成功可导出。
2. **Given** 初稿实测页数大于 N，且多出的末页内容很少（末页填充率 ≤ 稀疏阈值，默认 1/3），**When** 系统校准，**Then** 先在可读下限内收紧行距尝试压回 N 页；成功则不再剪枝。
3. **Given** 收紧行距后仍无法回到 N 页，**When** 系统继续校准，**Then** 触发 Agent 剪枝（仅删减/压缩弱相关内容，不编造），再测页；达标则成功。
4. **Given** 初稿实测页数等于 N，但末页明显偏空（填充率低于充实阈值），**When** 系统校准，**Then** 先尝试在可读区间内调整行距改善充实度；仍不足则 Agent 基于已有事实扩写或恢复未采用素材。
5. **Given** 初稿实测页数等于 N，且末页过空（填充率 < 稀疏阈值），**When** 系统校准，**Then** 跳过以行距凑满的路径，直接触发 Agent 扩写，并将该次初稿记为 Bad Case（预算低估）。
6. **Given** 多轮自动校准后仍无法同时满足「页数=N」与「末页充实」且不编造，**When** 进入引导态，**Then** 给出至少两条可执行建议（换紧凑主题、隐藏模块、减项目、补根简历、改选页数），禁用最终导出，且不得将实际页数伪报为 N。

---

### User Story 2 — 列表实际页数与打开后一致 (Priority: P1)

作为求职者，我在简历中心或岗位详情看到「目标 x 页 · 实际 x 页」时，希望与点开编辑器后的真实分页一致，避免被错误页数误导。

**Why this priority**: 当前用户可感知的信任问题；不修复则满页校准成果无法在列表侧体现。

**Independent Test**: 打开任意派生简历卡片，对比列表实际页数与编辑器预览页数；编辑保存后列表刷新一致。

**Acceptance Scenarios**:

1. **Given** 派生刚成功，**When** 用户查看列表，**Then**「实际页数」等于该次真实测量的页数，而非字符估算。
2. **Given** 用户打开派生简历后编辑导致分页变化，**When** 保存（或分页防抖持久化完成），**Then** 实际页数更新为当前预览页数，列表与页数控制面板同步。
3. **Given** 用户仅浏览打开未改内容，**When** 关闭编辑器，**Then** 系统不得因「打开」 alone 产生无意义写库；若打开时发现存量 actual 与当前预览不一致，可在保存路径或显式同步路径修正，但不得把只读浏览变成强制脏写风暴。

---

### User Story 3 — 导出以真实 PDF 页数为终裁 (Priority: P1)

作为求职者，我导出投递 PDF 时，希望文件页数严格等于目标页数；若预览与 PDF 短暂不一致，系统应拦截而不是放出错误页数文件。

**Why this priority**: REQ-055 已承诺的硬验收项；本特性不得削弱。

**Independent Test**: 对成功派生稿导出 PDF，用独立方式清点页数；故意制造预览/PDF 不一致时导出被拒。

**Acceptance Scenarios**:

1. **Given** 派生成功且实际页数=目标页数，**When** 导出 PDF，**Then** PDF 页数等于目标页数。
2. **Given** 渲染后 PDF 页数 ≠ 目标页数，**When** 尝试导出，**Then** 导出被拒绝，提示需重新校准或调整，并回写真实 PDF 页数到实际页数字段（或等价「过期/待校准」态）。

---

### User Story 4 — 校准决策可解释且可复盘 (Priority: P2)

作为求职者与运营方，我希望知道系统为何先调行距或调用 Agent，以及哪些「末页过空却页数已对」的初稿被记为 Bad Case。

**Why this priority**: 建立信任并支撑后续预算调参；不阻塞 P1 满页主路径，但为本特性闭环所需。

**Independent Test**: 完成一次含行距调整与一次含 Bad Case 的派生，检查页数报告中的决策痕迹与 Bad Case 记录字段完整性。

**Acceptance Scenarios**:

1. **Given** 校准过程执行了策略，**When** 查看派生运行/页数报告，**Then** 可见目标页数、实测页数、末页填充率、行距、策略序列与最终决策。
2. **Given** 发生「页数已等于目标但末页 < 稀疏阈值」的初稿，**When** 校准结束，**Then** 存在可检索的 Bad Case 记录（至少含目标、实测、末页填充率、主题、决策、结果）。

---

### Edge Cases

- 目标 1 页、实测 2 页、第 2 页仅约 1/3：先压行距；失败再剪枝。
- 目标 1 页、实测 2 页、第 2 页接近满页：少试行距，更快进入剪枝。
- 目标 2 页、实测 2 页、第 2 页约 2/3：可通过；不必强行扩写。
- 目标 2 页、实测 2 页、第 2 页约 1/2：先调行距改善充实度；失败再轻扩写。
- 目标 2 页、实测 2 页、第 2 页不足 1/3：直接扩写并记 Bad Case。
- 目标 3 页、实测 1 页：扩写路径；素材不足则引导补根简历或改小目标页，禁止编造。
- 目标 2 页、实测 4 页：强剪枝；达最大自动轮次仍失败则引导。
- 存在不可拆超高块（大表/大图）导致测量告警：进入引导，不盲目删光核心经历。
- 用户手改后页数偏离：保存回写实际页数；导出前再检，偏离则拦截。
- 预览 N 页但 PDF ≠ N：导出门禁失败并回写 PDF 实际页数。
- 行距已达可读下限仍超页：停止继续压行距，转剪枝或引导。
- 扩写无可用未采用素材且根简历无更多事实：引导态，不灌水。
- 派生任务取消/超时：保留可解释失败态，不写成功页数。
- 并发编辑与校准回写冲突：以明确的版本/所有权规则避免静默互相覆盖（继承既有简历保存语义）。

## Requirements *(mandatory)*

### Functional Requirements

- **FR-001**: 系统 MUST 在派生校准中使用与编辑器预览一致的真实分页测量，不得仅用字符长度估算作为「实际页数」真相。
- **FR-002**: 每次测量 MUST 产生：页数、末页填充率（及等价的已用高度/页内容高度）、当前行距；前 N−1 页视为满页，末页填充率用于决策。
- **FR-003**: 目标页数 N∈{1,2,3} 时，派生成功态 MUST 同时满足：实测页数 = N，且末页填充率 ≥ 充实阈值（默认 2/3，可配置）。
- **FR-004**: 当实测页数 > N 且末页填充率 ≤ 稀疏阈值（默认 1/3）时，系统 MUST 先在可读行距下限内尝试收紧行距；成功压回 N 页则不得无故剪枝。
- **FR-005**: 当收紧行距无法消除超页，或超页幅度大（例如多出 ≥2 页，或末页已较满仍超页）时，系统 MUST 触发 Agent 剪枝后再测。
- **FR-006**: 当实测页数 = N 且稀疏阈值 ≤ 末页填充率 < 充实阈值时，系统 MUST 先尝试在可读区间调整行距改善充实度；仍不足再触发 Agent 轻扩写。
- **FR-007**: 当实测页数 = N 且末页填充率 < 稀疏阈值时，系统 MUST 直接触发 Agent 扩写，并将该初稿标记为 Bad Case（预算低估类）。
- **FR-008**: 当实测页数 < N 时，系统 MUST 优先通过基于已有事实的扩写/恢复未采用素材追赶目标页；行距放松仅作为辅助，不得用无限拉大行距伪造「满页」。
- **FR-009**: Agent 剪枝与扩写 MUST NOT 编造无来源事实；无法达标时 MUST 进入引导态并禁用最终导出。
- **FR-010**: 行距调整 MUST NOT 低于主题/产品规定的可读下限。
- **FR-011**: 派生成功落库时，`实际页数` MUST 等于该次真实测量页数；引导态/失败态 MUST NOT 将实际页数伪报为等于目标页数。
- **FR-012**: 用户保存派生简历（或分页防抖持久化）时，系统 MUST 将当前预览页数同步到实际页数字段及列表展示来源。
- **FR-013**: 简历中心与岗位详情中的「目标/实际页数」MUST 反映 FR-011/FR-012 的权威值，并与打开后预览一致。
- **FR-014**: PDF 导出 MUST 以导出文件真实页数为终裁；与目标不符时 MUST 拒绝导出并回写真实 PDF 页数（或等价待校准态）。
- **FR-015**: 页数报告 MUST 记录目标、实测、末页填充率、行距、策略序列与最终决策，供用户侧页数面板与运营复盘使用。
- **FR-016**: 系统 MUST 持久化 FR-007 类 Bad Case 的最小字段集（目标、实测、末页填充率、主题、决策、结果、关联运行标识），以便后续评测与初稿预算调参。
- **FR-017**: 自动校准 MUST 有最大轮次上限；耗尽后进入引导态，不得无限循环调用模型。

### Governed Boundaries & Failure Semantics *(mandatory)*

- **Contract**: 派生运行状态继续区分成功 / 引导 / 失败；页数报告与导出门禁错误需可区分「测量失败」「页数不匹配」「充实度不达标」「素材不足不可行」。
- **Authorization**: 仅简历所有者可触发派生校准、保存回写与导出；模型侧仅处理该用户授权的根/派生内容。
- **Execution**: 校准在派生后台任务中执行；支持取消；轮次与模型调用有预算；重复投递不得产生重复成功派生行（继承既有幂等）。
- **External Effects**: 模型调用结果采纳前须通过事实/结构校验；失败可重试或降级到引导，不得半截脏写冒充成功。
- **Data Lifecycle**: page_report 与 Bad Case 随用户简历数据生命周期管理；删除用户/简历时需可清理或匿名化派生元数据（继承平台删除语义）。
- **Compatibility**: 旧派生行缺少填充率时，打开保存后按新测量回写；不以旧字符估算覆盖新测量。

### AI & Agent Safety Requirements *(mandatory when AI or Agent behavior is in scope)*

- **State**: 校准轮次、策略序列、测量快照属于派生运行可恢复状态的一部分；恢复后不得重复应用已成功的破坏性剪枝而不再测量。
- **Tools**: 剪枝/扩写工具只能改当前派生草稿；只读工具可查根简历与未采用素材；写工具风险为 R2。
- **Trust Boundary**: JD 与检索内容不得被当作可写入正文的权威事实来源；无根简历证据的要求只能进建议/追问。
- **Quality**: 决策表（行距 vs Agent）须有确定性单测；满页与稀疏末页场景须有样例验收；Bad Case 可统计。

### Key Entities *(include if feature involves data)*

- **页数测量结果**：页数、各页或至少末页的填充率、行距、测量版本/主题上下文。
- **页数报告**：一次派生校准的目标、测量序列、策略步骤、最终决策与结果。
- **派生简历页数状态**：目标页数、实际页数、是否可导出、是否待校准/过期。
- **充实度阈值**：稀疏阈值与充实阈值（默认 1/3 与 2/3）。
- **页数 Bad Case**：初稿页数已对但末页过空等可复盘记录。

## Success Criteria *(mandatory)*

### Measurable Outcomes

- **SC-001**: 在素材充足的验收样例集上，派生成功案例中预览页数与目标页数一致率达到 **100%**。
- **SC-002**: 同上成功案例中，末页填充率 ≥ 充实阈值（默认 2/3）的比例达到 **≥ 95%**；未达标者不得计为成功导出态。
- **SC-003**: 导出 PDF 页数与目标页数一致率达到 **100%**（不一致必须被拦截，计为正确拦截而非失败放过）。
- **SC-004**: 随机抽查已成功派生列表项，「列表实际页数 = 打开后预览页数」一致率达到 **100%**。
- **SC-005**: 对「目标 1 / 实测 2 / 末页 ≤ 1/3」类近失配样例，**≥ 80%** 在仅行距调整或行距+有限剪枝后达成成功，且优先路径为行距（有策略记录可证）。
- **SC-006**: 「页数已等于目标但末页 < 稀疏阈值」的初稿 **100%** 进入扩写路径并产生 Bad Case 记录（在功能开启后的新运行上统计）。
- **SC-007**: 素材不足无法满页时，**100%** 进入引导态且 **0%** 出现无来源编造正文；用户至少获得 2 条可执行下一步建议。

## Assumptions

- 「满满当当」操作化为：页数严格等于目标，且末页填充率 ≥ 充实阈值（默认 2/3）；不是每页 100% 填满。
- 稀疏阈值默认 1/3、充实阈值默认 2/3；可配置，但首发验收按默认值。
- 真实分页测量与当前编辑器预览分页规则对齐；PDF 终裁允许与预览存在已知字体/边距差异，但必须以 PDF 拒绝不一致导出。
- 可读行距下限沿用现有主题/产品规范，本特性不发明更小字号去硬凑页。
- 本特性编号为 **REQ-063**：用户口头所称 REQ-062 与已存在的「个人商业套餐与微信支付」（`specs/062-commercial-payment`）冲突，故顺延。
- 依赖既有一键派生流水线、编辑器分页能力与 PDF 导出门禁；本特性升级其测量与决策，而非重做选岗向导。
- Agent 扩写/剪枝仍受反编造与点数/预算治理（REQ-061）约束。
