# Feature Specification: 求职训练指挥台（工作台首页）

**Feature Branch**: `057-dashboard-home-optimize`

**Created**: 2026-07-10

**Status**: Draft

**Requirement ID**: REQ-057

**Input**: User description: "关注当前系统的首页问题（简历分支未对齐简历中心、今日待办无正确业务逻辑、活动显示内部事件名、提升建议/AI 建议定位混乱、首屏缓存待复查），并在深入思考 UI/UX、功能与缓存后，按推荐方案整理一份新的 REQ。"

## Background

工作台是登录后的默认首页。当前页面更像「指标墙 + 装饰建议」：简历区仍绑已退役的分支模型、今日待办不是今日面试、活动露出内部事件名、两套建议同源重复、首屏多域并行且无清晰加载/缓存纪律。

REQ-057 将工作台升级为 **求职训练指挥台**：

1. 先回答三句话——今天要面试吗？简历/投递卡在哪？下一步练什么？
2. 用 L0 / L1 / L2 分层注意力，保证首屏先可用。
3. 修齐数据源与文案可信度，并补上漏斗、准备包、冷启动、继续未完成等闭环入口。

**定位差异**：不做 Indeed 式岗位推荐 Feed，也不做实时聊天首页。InterCraft 优势是根/派生简历 + 模拟面试 + 岗位追踪；指挥台服务「准备与训练」。

### 产品模块

- 工作台（登录默认首页）
- 简历中心（REQ-055 根 / 派生）
- 求职追踪（面试日程与状态漏斗）
- 模拟面试（继续未完成会话）
- 活动流 / 能力与面试反馈（建议与洞察数据）

### 设计原则

1. **单一事实来源**：每面板只绑权威业务域，禁用退役模型。
2. **可行动**：能点就必须跳对地方或改对状态；禁止假勾选、假「实时」。
3. **人话优先**：中文业务文案，禁止内部事件码当标题。
4. **建议可验证**：有数据给可追溯建议，无数据给空态与下一步。
5. **三句话原则**：今日面试 → 资产/漏斗 → 下一步训练。
6. **分层注意力**：L0 行动 / L1 资产与建议 / L2 洞察；新面板默认 L2。
7. **产品验收优先**：缓存用「新鲜度 / 隔离 / 失效」描述，不把中间件写进成功标准。

### 首屏信息层级

| 层级 | 内容 | 加载期望 |
|---|---|---|
| **L0 关键** | 问候与情境句、下一场面试、主 CTA、今日面试列表；未完成冷启动时的三步条 | 决定「可用」；可先骨架 |
| **L1 重要** | 简历中心摘要、单一「下一步」建议、求职漏斗、面试准备包、继续未完成 | 骨架后填充；失败不挡 L0 |
| **L2 洞察** | 面试趋势、能力概览、最近活动 | 可延后；仅面板降级 |

### 已锁定产品决策

| 议题 | 决策 |
|---|---|
| 建议区 | **方案 A**：合并为单一「下一步」面板；去掉重复栏与无法兑现的「实时」暗示 |
| 今日待办交互 | **只展示 + 跳转**；不渲染勾选框 |
| 服务端摘要 | **纳入本 REQ**：一次返回 L0 + 约定 L1 摘要字段，分层新鲜度与写后失效 |
| 漏斗 / 准备包 / 冷启动 / 继续 | **纳入本 REQ**（P1–P2），非远期愿望 |
| 岗位 Feed / 实时 LLM 首页 | **明确不做** |

## User Scenarios & Testing *(mandatory)*

### User Story 1 - 今日指挥台 (Priority: P1)

作为求职者，我打开工作台应先看到「今天情境 + 下一场面试 + 明确主行动」，小屏也能点到主 CTA；指标若保留则必须可点且无假涨跌。

**Why this priority**: 指挥台的第一印象；当前主 CTA 在小屏隐藏，指标墙同权抢注意力。

**Independent Test**: 有今日面试 / 仅有明日面试 / 全新用户 三种账号打开工作台，核对情境句、主 CTA 与小屏可达性。

**Acceptance Scenarios**:

1. **Given** 用户今天有面试，**When** 进入工作台，**Then** L0 展示下一场面试的公司/职位与相对时间，主 CTA 导向该岗位准备或今日列表。
2. **Given** 今天无面试，**When** 进入工作台，**Then** 情境句诚实说明，主 CTA 按空态规则导向登记岗位 / 开始模拟面试 / 完善简历之一。
3. **Given** 仍展示统计数字，**When** 用户点击指标，**Then** 进入对应模块；无真实环比数据时不展示误导性涨跌。
4. **Given** 常见手机宽度，**When** 查看首屏，**Then** 至少有一个主行动入口可达。

---

### User Story 2 - 今日面试列表 (Priority: P1)

作为求职者，我希望「今日待办」只列出岗位追踪中**今天要面试**的岗位，点击进入岗位详情；无面试时看到清晰空态，且无假勾选。

**Why this priority**: 用户已定义的核心业务；当前数据源错误且不可用。

**Independent Test**: 今天 2 场、明天 1 场面试的账号，工作台仅显示 2 条且可跳转。

**Acceptance Scenarios**:

1. **Given** 岗位面试时间落在用户本地今天，**When** 打开工作台，**Then** 列表仅含这些岗位（公司、职位、时间），不含通用任务或其他日期。
2. **Given** 列表中有岗位，**When** 点击，**Then** 进入该岗位详情。
3. **Given** 今天无面试，**When** 打开工作台，**Then** 空态 + 求职追踪入口。
4. **Given** 「查看全部」存在，**When** 点击，**Then** 进入求职追踪（可带今日过滤），不得链回工作台自身。
5. **Given** 本阶段不做完成态，**When** 渲染列表，**Then** 不出现可勾选外观。

---

### User Story 3 - 简历区对齐简历中心 (Priority: P1)

作为求职者，我在工作台看到与简历中心一致的根/派生简历摘要，并能进入对应简历或简历中心。

**Why this priority**: 简历是核心资产；退役分支导致长期空白并污染「是否有简历」判断。

**Independent Test**: 有根+派生的账号与无简历账号分别验证摘要、链接与空态。

**Acceptance Scenarios**:

1. **Given** 简历中心有根与派生简历，**When** 打开工作台，**Then** 摘要展示类型标识与最近更新，展示上限内数量一致。
2. **Given** 点击某摘要，**When** 导航，**Then** 进入与简历中心一致的编辑/详情语义。
3. **Given** 无简历，**When** 打开工作台，**Then** 空态引导去简历中心；标题心智不得再是「简历分支」。
4. **Given** 建议或统计判断「是否有简历」，**When** 计算，**Then** 根或派生任一存在即视为有简历。

---

### User Story 4 - 单一「下一步」建议 (Priority: P1)

作为求职者，我只看到一个基于真实数据的「下一步」建议区：可验证、可跳转；无数据时给引导，不给假数据，不冒充实时 AI。

**Why this priority**: 双栏同源重复是信任杀手；合并后才能谈建议是否生效。

**Independent Test**: 无数据 / 单场面试 / 数据齐全三夹具；有简历中心简历时从不提示「无简历」。

**Acceptance Scenarios**:

1. **Given** 工作台建议相关区域，**When** 渲染，**Then** 仅一个建议面板，无两栏同文，无「实时」类无法兑现徽章。
2. **Given** 新用户缺少基础数据，**When** 查看建议，**Then** CTA 引导首场面试或补齐资产，无占位假公司/假分。
3. **Given** 至少 1 场完成面试但资产未齐，**When** 查看建议，**Then** 引用真实面试要点，并正确识别简历中心是否有简历。
4. **Given** 面试场次与简历/错题/投递足够，**When** 查看建议，**Then** 至少两类 hint 可追溯真实数据。
5. **Given** 建议 CTA，**When** 点击，**Then** 到达可继续完成动作的页面。

---

### User Story 5 - 首屏预算与摘要缓存 (Priority: P1)

作为求职者，我希望尽快看到 L0 并可操作；作为维护者，我希望有用户隔离的工作台摘要、分层新鲜度与写后失效，且二次进入可先缓存再刷新。

**Why this priority**: 无预算则指挥台再好也会「慢到不可用」；缓存必须与 L0 字段一起设计。

**Independent Test**: L2 延迟 3 秒时 L0 仍可用；跨用户不串读；改简历/今日面试后在新鲜度内更新；日切后今日列表正确。

**Acceptance Scenarios**:

1. **Given** L2 面板缓慢或失败，**When** 进入工作台，**Then** L0 可见可点，整页不空白。
2. **Given** 多面板依赖同一域，**When** 首屏加载，**Then** 无重复阻塞请求。
3. **Given** 工作台摘要缓存，**When** 用户 A/B 访问，**Then** 零串读；写简历/今日面试/活动/进行中会话后按规则失效或标明延迟。
4. **Given** 二次进入且网络较慢，**When** 打开工作台，**Then** 先出现上次可用 L0 摘要再后台刷新。
5. **Given** 本地日切，**When** 再次聚焦或刷新，**Then** 今日列表不含昨日面试。
6. **Given** 登录已确认会话，**When** 后台身份刷新，**Then** 不以全屏「正在校验」重挡工作台。

---

### User Story 6 - 求职漏斗 (Priority: P2)

作为求职者，我希望看到岗位状态简漏斗（投递中 / 面试中 / 待反馈），点击进入过滤后的求职追踪。

**Why this priority**: 回答「投递卡在哪」；与岗位追踪能力对齐。

**Independent Test**: 多状态岗位账号上计数一致，点击进入对应过滤。

**Acceptance Scenarios**:

1. **Given** 多状态岗位，**When** 打开工作台，**Then** 至少三段可点击汇总，计数与权威列表一致。
2. **Given** 点击某段，**When** 跳转，**Then** 到达带该状态过滤的求职追踪。
3. **Given** 无岗位，**When** 打开工作台，**Then** 空态 + 登记入口，无伪造占比。

---

### User Story 7 - 面试准备包 (Priority: P2)

作为今日或下一场有面试的求职者，我希望从工作台一键到达岗位 + 关联派生简历（或去派生）+ 可选模拟入口。

**Why this priority**: InterCraft 差异化闭环的轻量入口。

**Independent Test**: 有/无派生简历的今日面试岗位，准备路径分别正确。

**Acceptance Scenarios**:

1. **Given** 今日或下一场岗位已关联派生简历，**When** 打开准备入口，**Then** 可达岗位详情并打开该派生简历。
2. **Given** 无派生简历，**When** 打开准备入口，**Then** 引导一键派生或简历中心，不静默失败。
3. **Given** 现有面试模式支持带岗位上下文，**When** 提供模拟入口，**Then** 可进入；不支持则隐藏，不死链。

---

### User Story 8 - 新用户三步冷启动 (Priority: P2)

作为新用户，我希望用「完善简历 → 登记岗位 → 首场模拟面试」进度条代替多块空态堆叠。

**Why this priority**: 降低「不知道先点哪」的冷启动流失。

**Independent Test**: 新账号见三步；完成一步进度更新；三步完成后退出 L0。

**Acceptance Scenarios**:

1. **Given** 缺少简历、岗位、已完成面试任一项，**When** 打开工作台，**Then** 显示最多三步进度，当前步高亮。
2. **Given** 完成某步，**When** 回到工作台，**Then** 该步完成，下一步成主 CTA。
3. **Given** 三步均完成，**When** 打开工作台，**Then** 冷启动条不再占 L0 主位。

---

### User Story 9 - 继续未完成 (Priority: P2)

作为中断过流程的求职者，我希望看到可恢复的进行中模拟面试（及已有稳定恢复态的派生任务）并一键继续。

**Why this priority**: 中断恢复是留存关键路径。

**Independent Test**: 进行中会话出现「继续」；结束后消失。

**Acceptance Scenarios**:

1. **Given** 可恢复的进行中模拟面试，**When** 打开工作台，**Then** L1 展示「继续面试」并进入该会话。
2. **Given** 派生任务有稳定可恢复态，**When** 打开工作台，**Then** 可展示对应继续入口；否则不展示。
3. **Given** 无可恢复项，**When** 打开工作台，**Then** 不渲染空的继续区块。

---

### User Story 10 - 最近活动可读 (Priority: P2)

作为求职者，我在最近活动中看到中文业务摘要，而不是 `job_created` 之类内部名。

**Why this priority**: L2 洞察仍影响信任；修复成本低、体感强。

**Independent Test**: 触发岗位创建/状态变更/面试完成等后，标题均为中文。

**Acceptance Scenarios**:

1. **Given** 已知活动类型，**When** 展示，**Then** 主标题为中文，不以内部事件码为唯一标题。
2. **Given** 有公司/职位等上下文，**When** 展示详情，**Then** 优先用这些字段。
3. **Given** 未知类型，**When** 展示，**Then** 中性中文回退，不暴露原始 type。
4. **Given** 活动可关联对象，**When** 点击，**Then** 跳转该对象；否则不可点。

### Edge Cases

- 「今日」按用户本地日历日；日切后下次聚焦/刷新更新；打开中页面不得永久卡在昨日。
- 仅有日期无时刻：仍按本地日归属；无法解析的时间不进今日。
- 面试已过点但状态未改：可标「已过点」并引导更新状态。
- 仅有派生、根缺失：不崩溃，展示可得简历并提示去简历中心。
- 漏斗未知状态：归「其他」或排除，不整页失败。
- 建议/L2 失败：局部降级，不影响 L0。
- 缓存脏读窗口：敏感域（今日面试、简历摘要）优先正确，或提供刷新。
- 未登录/会话过期：任何缓存路径都不得露出用户数据。
- 小屏空间不足：优先 L0 与建议；趋势/能力可更靠下。

## Requirements *(mandatory)*

### Functional Requirements

#### L0 指挥台与今日面试

- **FR-001**: 工作台 L0 MUST 包含基于真实数据的情境句、主 CTA、今日面试列表（或空态）。
- **FR-002**: 今日面试列表 MUST 仅含岗位追踪中面试时间属于用户本地今天的岗位。
- **FR-003**: 今日面试条目 MUST 展示公司、职位、时间，MUST 可导航至岗位；「查看全部」MUST 导向求职追踪。
- **FR-004**: 本阶段 MUST NOT 渲染今日列表的可勾选外观。
- **FR-005**: 主 CTA 在常见手机宽度下 MUST 可达。
- **FR-006**: 统计指标若展示，MUST 可导航至对应模块；无真实增长数据时 MUST NOT 展示误导性涨跌。

#### 简历中心对齐

- **FR-007**: 简历面板 MUST 使用简历中心根/派生模型，MUST NOT 使用已退役简历分支列表。
- **FR-008**: 简历摘要 MUST 含类型标识与最近更新，展示上限建议 5；空态 MUST 引导简历中心。
- **FR-009**: 任何「是否有简历」判断 MUST 与简历中心一致（根或派生任一即可）。

#### 单一下一步建议

- **FR-010**: 工作台 MUST 仅渲染一个建议面板（方案 A），MUST NOT 两栏同文。
- **FR-011**: 建议 MUST 基于真实面试/简历中心/错题/投递/能力数据，MUST NOT 使用占位假数据。
- **FR-012**: 建议档位 MUST 可测试；CTA MUST 指向可完成动作的真实页面。
- **FR-013**: MUST NOT 展示无法兑现的「实时」类 AI 暗示；MUST NOT 用纯前端启发式冒充 AI 产物。
- **FR-014**: 建议数据源部分失败时 MUST 面板降级，不拖垮整页。

#### 漏斗 / 准备包 / 冷启动 / 继续

- **FR-015**: 工作台 MUST 展示至少三段可点击的岗位状态漏斗摘要；无岗位时走空态。
- **FR-016**: 对今日或下一场面试，工作台 MUST 提供准备路径：岗位详情 + 关联派生简历或去派生引导；模拟入口仅在支持岗位上下文时展示。
- **FR-017**: 未完成冷启动时 MUST 展示最多三步（简历 → 岗位 → 首场面试），完成后退出 L0 主位。
- **FR-018**: 存在可恢复进行中模拟面试时 MUST 展示「继续」；无则隐藏。

#### 最近活动

- **FR-019**: 已知活动类型 MUST 有中文主标题映射；未知类型 MUST 中性中文回退。
- **FR-020**: 活动详情 MUST 优先业务上下文字段；MUST NOT 以内部事件码为唯一标题。
- **FR-021**: 活动写入方 SHOULD 提供稳定可读摘要；缺失时展示层仍须满足 FR-019～FR-020。

#### 首屏性能与缓存

- **FR-022**: 工作台 MUST 按 L0/L1/L2 加载；L0 为可用判定；L1/L2 可独立失败。
- **FR-023**: 首屏 MUST 消除同一关键域的重复阻塞请求。
- **FR-024**: 系统 MUST 提供按用户隔离的工作台摘要（一次返回 L0 及约定 L1 字段）。
- **FR-025**: 摘要缓存 MUST 分层新鲜度：今日面试与进行中会话为短；漏斗与活动为中；能力与趋势为长。
- **FR-026**: 摘要 MUST 在简历变更、今日面试相关岗位变更、新活动写入、进行中面试会话变更时失效或在约定窗口内可见更新。
- **FR-027**: 客户端 MUST 支持二次进入时先展示可用缓存再后台刷新；日切后今日域 MUST 重取。
- **FR-028**: 已确认登录会话 MUST NOT 因后台身份刷新被全屏鉴权遮罩重挡（对齐 REQ-037 最小集）。

#### 周边配置

- **FR-029**: 侧栏/顶栏若展示今日面试相关计数，MUST 与工作台今日列表同源规则。
- **FR-030**: 新工作台面板默认层级 MUST 为 L2，除非产品显式批准进入 L0/L1。
- **FR-031**: 各面板 MUST 有骨架/局部加载与局部错误重试，MUST NOT 整页空白。
- **FR-032**: 工作台关键控件 MUST 具备可见焦点与可读对比；动效不得作为唯一状态传达。

### Key Entities

- **求职训练指挥台**：工作台首页的产品形态（L0/L1/L2）。
- **今日指挥台**：情境句 + 下一场面试 + 主 CTA。
- **今日面试条目**：面试日为今天的岗位摘要。
- **简历摘要**：根/派生简历只读卡片。
- **下一步建议**：单一、可验证的分档建议。
- **求职漏斗摘要**：岗位状态可点击计数。
- **面试准备包**：岗位 + 派生简历 + 可选模拟入口。
- **冷启动进度**：三步完成态。
- **可恢复事项**：进行中面试会话等。
- **活动条目**：中文可读的近期事件。
- **工作台摘要**：按用户隔离的首屏聚合快照。

## Success Criteria *(mandatory)*

### Measurable Outcomes

- **SC-001**: 有今日面试账号中，内部任务脚本下 ≥95% 能在 10 秒内指出下一场面试是谁、从哪准备。
- **SC-002**: 「今天 2 / 明天 1」夹具下今日列表恰好 2 条，点击到达对应岗位；0 假勾选。
- **SC-003**: 简历区与简历中心在展示上限内数量一致率 100%；0「简历分支」主心智文案。
- **SC-004**: 建议面板数量为 1；内容重复率 0；有简历中心简历时错误「无简历」提示 0 次。
- **SC-005**: L2 延迟 3 秒时 L0 仍可导航；登录后不被二次全屏鉴权重挡。
- **SC-006**: 摘要缓存跨用户串读 0；写今日面试/简历后在约定新鲜度内可见更新（或明确延迟提示）。
- **SC-007**: 小屏主 CTA 可达率 100%。
- **SC-008**: 漏斗三段点击均可到达对应过滤列表；无岗位时无伪造占比。
- **SC-009**: 冷启动三步账号无需离开工作台即可点到当前步 CTA；完成后引导退出 L0。
- **SC-010**: 进行中面试「继续」出现率 100%，结束后残留 0。
- **SC-011**: 抽检 20 条可见活动，0 条以内部事件码为唯一主标题。
- **SC-012**: 二次进入慢网下 L0 无整页白屏；日切后今日列表不含昨日面试。

## Assumptions

- 工作台即现有登录默认首页，不另建第二套 Home。
- 「今日」以用户本地日历日验收。
- 今日列表权威源是岗位面试时间，不是通用任务表。
- 简历中心（REQ-055/056）已是简历权威模型。
- 漏斗三段映射在 plan 阶段对齐现有岗位状态机（投递中 / 面试中 / 待反馈为产品名）。
- 准备包不依赖 REQ-053 深度调研报告；无报告仍可走岗位+简历路径。
- 018 去假数据精神保留，数据源全面改为简历中心。
- 指标卡可降权或并入漏斗/情境，不强制删除所有数字。

## Out of Scope
